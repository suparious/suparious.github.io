Do not modify resume.json or the live site for individual job applications. The resume is a stable base document.

**Why:** Shaun plans to apply to 5+ jobs per day (~90+/month). Tweaking the resume per application is unsustainable at that volume and would constantly change the live site.

**How to apply:** Customization goes in the cover letter, not the resume. May create 2-3 resume *variants* for different role types, but these are static files, not per-job edits.
